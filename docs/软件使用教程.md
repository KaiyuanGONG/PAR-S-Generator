# PAR-S Generator 使用教程

适用版本：`v0.3.0`

## 1. 总览

当前软件左侧主栏只有两个核心模块：

1. `Generate`
2. `Simulate`

左下角工具区提供：

1. `Settings`
2. `About`

也就是说，原来的 `Review` 已经不再是单独页面，而是被整合进 `Generate`。

此外，这一版还支持三种界面语言：

- `English`
- `中文`
- `Français`

语言切换会立即作用到当前界面，不需要重启。

---

## 2. Generate

`Generate` 是现在最核心的工作区。它内部有两个分页：

1. `Preview`
2. `Batch Monitor`

### 2.1 Preview

这个分页负责三件事：

1. 调参数
2. 看单例预览
3. 设置并启动批量生成

### 2.2 左侧参数区

参数现在分成四组：

1. `Volume`
2. `Liver Geometry`
3. `Tumors`
4. `Activity`

统一交互逻辑如下：

- 默认状态下只能在推荐范围内通过滑块调节
- 勾选 `Advanced` 后，允许超出推荐范围手动调整
- 即使开启 `Advanced`，也仍然受硬安全范围保护，不会放任参数把算法弄崩
- 每个参数都带有一句 tooltip，鼠标悬停即可看到简短解释

### 2.3 Volume

这里重点控制：

- `Matrix`
- `Voxel size`

默认提供三组常用预设：

- `96 / 5.89 mm`
- `128 / 4.42 mm`
- `160 / 3.54 mm`

设计含义是：在保持大致腹部覆盖范围相近的前提下，改变采样精细度。

#### 重要说明

这里现在明确采用以下语义：

- 肿瘤大小按物理 `mm` 定义
- 修改 `matrix` 或 `voxel size` 时，不会自动按比例缩放肿瘤直径元数据

也就是说，肿瘤不是“跟着体素一起变大的 UI 视觉块”，而是“物理尺寸固定、体素表示方式变化”的对象。

### 2.4 Liver Geometry

你可以调整：

- `Scale jitter`
- `Rotation jitter`
- `Global shift range`
- `Target left ratio`
- `Smoothing sigma`

这些参数都带有推荐范围和硬安全范围。默认滑块只在合理范围内移动；勾选 `Advanced` 才能进一步放开。

#### 左叶比例显示修复

这一版已经修正显示错误：

- `0.35` 会显示为 `35.0%`
- 不会再误显示成 `0.35%`

### 2.5 Tumors

这一组现在明确区分了预览和批量：

- 批量仍然使用 `Min tumors / Max tumors`
- 单例预览可以勾选 `Use exact count`，直接指定精确肿瘤数量

此外还支持：

- `Contrast min / max`
- `Tumor style`

肿瘤样式有三种：

1. `Ellipsoid`
2. `Spiculated`
3. `Random`

界面里也会提示当前算法使用的物理尺寸分箱：

- `10–20 mm`
- `20–40 mm`
- `40–60 mm`

### 2.6 Activity

这一组主要是：

- `Total counts`
- `Residual BG`
- `Perfusion mode`

灌注模式可选：

1. `Whole Liver`
2. `Tumor Only`
3. `Left Only`
4. `Right Only`
5. `Random`

同时要注意：

- Python 侧不会再伪造 `PSF` 模糊参数
- `PSF` 仍然由 SIMIND 负责

### 2.7 右侧预览区

右侧是 `2 x 2` 的预览布局：

1. 轴向
2. 冠状
3. 矢状
4. 指标卡面板

第四块现在不再重复显示一堆和底部重复的文字，而是改成了可视化指标卡，集中展示：

- 肝脏体积
- 左叶比例
- 肿瘤数
- 总计数
- 灌注模式
- 肿瘤样式
- 几何信息

可切换 overlay：

- `Liver + Tumors`
- `Tumors`
- `Liver`
- `Contours`
- `No Overlay`

同时保留 `3D Surface` 视图。

### 2.8 底部批量设置条

原来底部醒目的可视化指标区域已经改成真正的批量设置条，包含：

- `Number of cases`
- `Use fixed seed`
- `Global seed`
- `Output directory`
- `Start Batch`

#### 预览和批量的关系

这里给出明确答案：

- 预览和批量共享同一套参数配置
- 但预览结果本身不参与批量序列
- 预览只是帮你验证当前参数设置是否合理

当前种子策略是：

- 预览使用 `case_id = 0`
- 批量使用 `case_id = 1..N`
- 如果启用固定种子，则每个案例的种子为 `global_seed + case_id`

所以：

- 预览不会消耗批量序列
- 批量可以稳定复现

## 3. Batch Monitor

`Generate` 的第二个分页是 `Batch Monitor`。

这里整合了原来 `Review` 的主体功能：

- 实时进度条
- ETA
- Elapsed
- 统计卡片
- 统计图表
- case 表格
- 日志
- 加载已有 `batch_summary.json`

### 3.1 Load Existing Summary

现在这个功能不再只是把 JSON 打印出来，而是会真正回填：

- 统计卡片
- 图表
- case 表格

### 3.2 注意

`.a00` 的可视化已经从这里移除，不再和仿真模块重复。

---

## 4. Simulate

`Simulate` 现在是唯一的仿真与 `.a00` 查看入口。

界面被整理成四步：

1. `Step 1: Raw Binary Export`
2. `Step 2: SIMIND Configuration`
3. `Step 3: Script or Run`
4. `Step 4: Visual Check`

### 4.1 Step 1: Raw Binary Export

这里把 `Generate` 生成出来的 `case_*.npz` 导出成 SIMIND 需要的原始二进制文件。

输出文件是：

- `case_XXXX_act_av.bin`
- `case_XXXX_atn_av.bin`

注意：

- 当前软件导出的是真实 `.bin`
- 不是以前文档里提到的 `.h33/.i33`

### 4.2 Step 2: SIMIND Configuration

这里设置：

- `simind.exe`
- `.smc`
- SIMIND 输出目录

软件会自动继承：

- `Generate` 中的 phantom 输出目录
- `Settings` 中保存的默认 `simind.exe`
- `Settings` 中保存的默认 `.smc`

### 4.3 Step 3: Script or Run

这里有两个动作：

1. `Generate .bat Script`
2. `Run SIMIND Now`

### 4.4 Step 4: Visual Check

右侧预览区现在统一叫：

- `SPECT Preview`

仿真完成后，软件会自动加载第一个 `.a00` 文件供快速检查。

这也是整个软件里唯一保留的 `.a00` 查看器。

---

## 5. Settings

`Settings` 不再和 `About` 混在一个滚动页面里，而是单独的设置弹窗。

设置项包括：

- 默认 `simind.exe`
- 默认 `.smc`
- 默认输出目录
- 主题
- 语言
- `Auto-save config on batch start`

### 5.1 设置持久化

这一版不再依赖 `QSettings`。

改为保存到 JSON 文件：

- `%APPDATA%/PAR-S Generator/settings.json`

这样能避免你当前环境下 `QSettings.AccessError` 的问题。

### 5.2 自动保存配置

如果启用 `Auto-save config on batch start`，批量开始时会在输出目录自动写入：

- `last_batch_config.json`

---

## 6. About

`About` 已经从设置页里拆出来，放在左下角工具区单独打开。

这样设置页不再因为加了说明文字而出现不必要的滚动条。

---

## 7. 约束与校验

现在软件会在真正启动长任务前做前置校验。

### 7.1 直接阻断的情况

例如：

- `Min tumors > Max tumors`
- `Contrast min > Contrast max`
- 输出目录为空
- `simind.exe` 路径错误
- `.smc` 路径错误
- `npz` 目录不存在
- 体积不是立方体
- bundled `ge870_czt.smc` 与当前 geometry 不匹配

### 7.2 警告但允许继续的情况

例如：

- 参数超出推荐范围，但仍未超出硬安全范围
- 修改了 matrix / voxel，导致默认 bundled `.smc` 不再匹配

## 8. 输出文件

### 8.1 体模输出

```text
output/syn3d/
├── case_0001.npz
├── case_0001_meta.json
├── case_0002.npz
└── batch_summary.json
```

### 8.2 Binary 输出

```text
output/interfile/
├── case_0001_act_av.bin
├── case_0001_atn_av.bin
├── case_0002_act_av.bin
└── ...
```

### 8.3 SIMIND 输出

```text
output/simind/
├── case_0001.a00
├── case_0001.h00
├── case_0001.res
└── ...
```

## 9. 关键提醒

### 9.1 默认 bundled `.smc` 仍然只适配参考几何

当前 bundled 的 `ge870_czt.smc` 仍然按以下几何设计：

- `128 x 128 x 128`
- `4.42 mm`

如果你修改了 matrix 或 voxel，就必须自己提供匹配的 `.smc`。

### 9.2 photon histories 不由 UI 控制

这一版已经彻底移除了假的 photon slider。

真正的 photon histories 仍由 `.smc` 决定。

---

## 10. 建议使用顺序

推荐你按这个顺序使用：

1. 在 `Generate > Preview` 调参数并确认单例预览
2. 在同页底部设置批量数量、种子和输出目录
3. 点击 `Start Batch`，切到 `Batch Monitor` 看统计
4. 批量结束后进入 `Simulate`
5. 先做 `Raw Binary Export`
6. 再配置 `.smc` 和输出路径
7. 最后运行 SIMIND，并在 `SPECT Preview` 中检查 `.a00`
