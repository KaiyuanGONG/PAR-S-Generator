## 1. 启动软件

```bash
# 激活虚拟环境
.venv\Scripts\activate

# 启动
python main.py
```

首次启动约需 10–30 秒（PyQt6 + NumPy 加载）。

软件启动时自动检测：
- `simind/simind.exe` → 自动填入 Simulation 页面
- `simind/ge870_czt.smc` → 自动填入 .smc 路径

## 2. 快速验证流程（首次使用）

### 步骤 1：验证体模生成（约 3–5 秒）

1. 启动后默认停在 **Phantom** 页面
2. 保持默认参数不变，点击 **"Preview Single Case"**
3. 右侧出现三平面切片图，右下角信息面板应显示：
   - Liver volume：900–2200 mL
   - **Left lobe ratio：20%–50%**（如显示 0% 说明环境未更新，需重启）
   - Perfusion mode：非空字符串

### 步骤 2：批量生成（约 5 秒/case）

1. 设置 **Output directory** 为你想要的路径（如 `D:\data\syn3d`）
2. 设置 **Number of cases = 1**（先测试单个）
3. 切换到 **Results 页面**，点击 **"▶ Start Batch Generation"**
4. 完成后在输出目录看到 `case_0001.npz` + `case_0001_meta.json`

> **注意**：编号从 1 开始（`case_0001`），这是设计行为。

### 步骤 3：格式转换（约 1 秒/case）

1. 切换到 **Simulation 页面**
2. Step 1 — npz directory：选择含 `.npz` 文件的目录
   - 选目录后立即显示找到的文件数（如 `Found 1 .npz file(s)`）
3. Binary output：指定输出目录（如 `D:\data\binary`）
4. 点击 **"Convert All Cases"**
5. 验证输出：`case_0001_act_av.bin` 和 `case_0001_atn_av.bin`，各 **8,388,608 字节**（8 MB）

### 步骤 4：SIMIND 仿真（约 4 分钟/case）

1. Step 2：确认 simind.exe 和 ge870_czt.smc 已自动填入
2. 设置 Output dir（如 `D:\data\simind_out`）
3. 点击 **"Generate .bat Script"**（先检查脚本内容）
4. 确认无误后点击 **"▶ Run SIMIND Now"**
5. Console Output 中看到 SIMIND 逐投影进度
6. 完成后自动跳转到 **SIMIND Output** 标签页

### 步骤 5：验证仿真结果

1. 切换到 Results → **SIMIND Output** 标签页
2. 点击 **"Load .a00 File"**，选择 `case_0001.a00`
3. 左侧显示投影图像，右侧显示 sinogram
4. 拖动滑块浏览 60 个投影角度
5. Sinogram 应显示类似弦状弧线（点源→正弦曲线；均匀肝脏→宽弧带）

## 4. Phantom 页面 — 体模参数配置与预览

### 4.1 左侧参数面板

#### VOLUME

| 参数 | 默认值 | 说明 |
|:---|:---|:---|
| Matrix (NxNxN) | 128 | 体素矩阵边长，必须与 .smc 中 Index-76/78/81 一致 |
| Voxel size (mm) | 4.42 | GE NM/CT 870 标准值，与 .smc Index-28/31 一致 |

#### LIVER GEOMETRY

| 参数 | 默认值 | 说明 |
|:---|:---|:---|
| Scale jitter | 0.10 | 肝脏大小随机变化幅度（±10%） |
| Rotation jitter (°) | 5.0 | 旋转随机变化 |
| Global shift range | 0.05 | 整体位移随机范围 |
| Target left ratio | 0.35 | 目标左叶体积比例（二分法精确控制，精度 <0.012%） |
| Smoothing σ (px) | 1.2 | 肝脏轮廓高斯平滑 |

#### TUMORS

| 参数 | 默认值 | 说明 |
|:---|:---|:---|
| Min/Max tumors | 0 / 5 | 每个 case 肿瘤数量范围 |
| Contrast min/max | 4.0 / 8.0 | 肿瘤相对肝脏的活度对比度 |

肿瘤形态随机选择：spiculated（毛刺）/ ellipsoid / superellipsoid / noise_threshold

#### ACTIVITY

| 参数 | 默认值 | 说明 |
|:---|:---|:---|
| Total counts (×10⁴) | 8.0 | 活度图总计数（=80,000）|
| PSF σ (px) | 2.5 | UI 保留但不生效，PSF 由 SIMIND 内部处理 |
| Residual BG | 0.05 | 灌注缺损区的残余活度比例 |

> **关于 Total counts**：若 SIMIND 仿真后 .a00 中计数为 0，需将此值提高至 1×10⁷（1000.0×10⁴），使每个肝脏体素有足够的放射源强度。

灌注模式（随机）：
- Whole Liver (5%) / Tumor Only (25%) / Left Only (35%) / Right Only (35%)

#### BATCH GENERATION

| 参数 | 默认值 | 说明 |
|:---|:---|:---|
| Number of cases | 10 | 批量 case 数 |
| Global seed | 42 | 随机种子（勾选 Use fixed seed 时生效） |
| Output directory | output/syn3d | .npz 文件保存目录 |

> **重要**：Output directory 在这里设置后立即生效，无需点击 Preview 再去 Results 页面。

### 4.2 按钮

| 按钮 | 功能 |
|:---|:---|
| **Preview Single Case** | 生成预览（不保存到磁盘），用于快速验证参数 |
| **Save Config** | 保存当前参数为 JSON |
| **Load Config** | 从 JSON 恢复参数 |

### 4.3 右侧预览区

**Multi-Plane 标签**：Axial (Z) / Coronal (Y) / Sagittal (X) 三平面 + 右下信息面板

| 信息面板字段 | 含义 |
|:---|:---|
| Volume shape | 矩阵尺寸 |
| Voxel size | 固定 4.42 mm |
| Liver volume | 肝脏体积 (mL) |
| **Left lobe ratio** | 二分法分割结果（应在 20%–50% 之间） |
| No. of tumors | 实际植入肿瘤数 |
| **Perfusion mode** | 本次随机选择的灌注模式 |
| Total counts | 活度图总计数 |

**3D Surface 标签**：蓝色透明体 = 肝脏，红色 = 肿瘤（需安装 scikit-image）

**Channel 切换**：Activity Map / μ-map (Attenuation)

**Liver Overlay**：开启时绿色高亮肝脏范围。查看肿瘤时建议**关闭 Overlay**，肿瘤对比度更清晰。

## 5. Results 页面 — 批量生成与统计

### 5.1 操作

1. 在 **Phantom 页面**设置好参数（特别是 Output directory 和 Number of cases）
2. 切换到 **Results 页面**，参数自动读取（每次点击 Start 时都会读最新值）
3. 点击 **"▶ Start Batch Generation"**

> 不需要先在 Phantom 页面点 Preview，直接在 Results 页面点 Start 即可。

### 5.2 进度区

- 进度条：已完成/总数
- ETA 和 Elapsed 实时更新
- Case Table 实时追加行

### 5.3 统计卡片

| 卡片 | 含义 |
|:---|:---|
| Completed | 已完成/总数 |
| Liver Vol. Mean | 平均肝脏体积 (mL) |
| Left Ratio Mean | 平均左叶比例（应接近 0.350） |
| Avg Tumors/Case | 平均每 case 肿瘤数 |
| Total Tumors | 所有 case 总肿瘤数 |
| Avg Gen. Time | 平均每 case 耗时 (s) |

### 5.4 标签页

| 标签 | 内容 |
|:---|:---|
| **Statistics Charts** | 6 张分布图（肝体积、左叶比例、肿瘤数、肿瘤直径、灌注模式、生成时间） |
| **Case Table** | 每 case 详细数据，含 Case ID（从 0001 开始）、种子、灌注模式等 |
| **Log** | 生成日志 |
| **SIMIND Output** | .a00 投影可视化（见第 7 节）|

### 5.5 输出文件

```
output/syn3d/
├── case_0001.npz          ← 体模数据
├── case_0001_meta.json    ← 元数据（肿瘤参数）
├── case_0002.npz
├── ...
└── batch_summary.json     ← 统计汇总
```

## 6. Simulation 页面 — 格式转换与 SIMIND 仿真

### 6.1 STEP 1 — FORMAT CONVERSION (npz → .bin)

| 字段 | 说明 |
|:---|:---|
| npz directory | 含 `case_XXXX.npz` 的目录 |
| Binary output | 输出 `.bin` 文件目录（选目录后立即显示找到的文件数） |
| Voxel size | 应与 Phantom 页面一致（默认 4.42 mm） |

点击 **"Convert All Cases"** 后输出：

```
output/binary/
├── case_0001_act_av.bin   ← 活度图（SIMIND 通过 /FS:case_0001 读取）
├── case_0001_atn_av.bin   ← 衰减图（SIMIND 通过 /FD:case_0001 读取）
├── case_0002_act_av.bin
└── ...
```

文件格式：float32，C-order (Z,Y,X)，无头文件，大小恒为 **8,388,608 字节**（128³×4）。

> **命名约定**：后缀 `_av` 是 SIMIND XcatBinMap（Index=-7）的强制要求，不可更改。

### 6.2 STEP 2 — SIMIND CONFIGURATION

| 字段 | 说明 |
|:---|:---|
| simind.exe | 启动时自动检测 `simind/simind.exe` |
| .smc config | 自动检测 `simind/ge870_czt.smc` |
| Output dir | SIMIND 输出目录（`.a00`、`.mhd`、`.res`、`.spe` 等） |
| Photon histories | 仅显示用，实际光子数由 .smc 中 Index-26 控制 |

### 6.3 STEP 3 — GENERATE & RUN

| 按钮 | 功能 |
|:---|:---|
| **Generate .bat Script** | 生成脚本后先检查内容再运行 |
| **▶ Run SIMIND Now** | 自动生成脚本后立即运行，Console Output 实时显示 |
| **■ Stop** | 终止当前进程 |

**生成的 .bat 脚本逻辑：**
```batch
# 将 .smc 复制到 binary 目录（SIMIND 解析路径时把反斜杠当作参数分隔符，
# 用相对路径 /FS:case_0001 可避免此问题）
copy "simind\ge870_czt.smc" "output\binary\" /Y > nul

pushd "output\binary"        ← cd 到 binary 目录
simind.exe ge870_czt "D:\output\simind\case_0001" /FS:case_0001 /FD:case_0001
popd
```

**关键约束**：
- `/FS:` 和 `/FD:` 只传 stem（`case_0001`），**不加路径、不加 `_act_av`**
- SIMIND 自动在当前目录找 `case_0001_act_av.bin` 和 `case_0001_atn_av.bin`
- 输出 stem（第2参数）用绝对路径，SIMIND 正确处理

**Console Output 颜色含义：**
- 蓝色 `[INFO]` — 一般信息
- 绿色 `[OK]` — 成功
- 红色 `[ERROR]` — 需处理的错误
- 橙色 `[WARN]` — 警告

**SIMIND 正常输出文件（每个 case）：**

| 文件 | 说明 |
|:---|:---|
| `.a00` | 主窗投影（float32，60×128×128 = 3,932,160 字节） |
| `.mhd` | MetaImage 格式头文件 |
| `.res` | 仿真摘要统计 |
| `.spe` | 能谱数据 |

## 7. SIMIND Output 标签 — 投影与 Sinogram 可视化

Results 页面的第 4 个标签页，用于直观验证 SIMIND 仿真结果。

SIMIND 仿真完成后会**自动跳转到此标签**。

### 7.1 加载文件

点击 **"Load .a00 File"** → 选择任意 `.a00` 文件。

软件自动检测维度（支持 128×128、64×64、256×256 矩阵）。

加载后顶部显示统计信息：
```
60 proj  |  128×128  |  Max: 1234.5  |  Total: 5.231e+06
```

如果 **Total 接近 0**，说明仿真没有产生计数——参见第 12 节排查。

### 7.2 投影查看器（左侧）

- 显示单个投影（一个角度的探测器图像）
- 拖动 **Projection 滑块** 在 1–60 个投影间切换
- 右上角显示当前投影编号和对应角度（从 180° 开始，每步 6°）
- 图像用 pyqtgraph 显示，**支持鼠标滚轮缩放**和拖拽查看细节

**正常投影外观：**
- 整体呈椭圆形（肝脏投影轮廓）
- 中央区域亮度较高（活度集中区）
- 随投影角度变化，形状和亮度分布应平滑演变
- 肿瘤区域可能出现局部亮斑

### 7.3 Sinogram 查看器（右侧）

Sinogram 显示**所有角度**对同一个探测器行的计数分布：
- X 轴 = 探测器列位置（0–127）
- Y 轴 = 投影角度（0–59，即 180°–534°）
- 拖动 **Row 滑块** 选择探测器行（通常选中间行 64 最有信息量）

**Sinogram 判读：**

| 外观 | 解释 |
|:---|:---|
| 中央呈现弧形亮带 | 正常——肝脏活度在旋转过程中形成的正弦形迹 |
| 完全空白/全黑 | 仿真无计数，检查活度图值和 SIMIND 配置 |
| 单条亮线（不随角度变化） | 探测器几何错误，源被固定在一个位置 |
| 随机噪声点 | 统计噪声正常，增大 Index-26 光子数可改善 |

## 9. 完整工作流程（端到端）

```
① Phantom 页面
   设置参数（Matrix=128, Voxel=4.42mm, Total counts=8e4, N cases=100）
   设置 Output directory = D:\data\npz
   点击 Preview 确认外观合理
        ↓

② Results 页面
   点击 ▶ Start Batch Generation
   → 输出：D:\data\npz\case_0001.npz ... case_0100.npz
   → D:\data\npz\batch_summary.json
        ↓

③ Simulation 页面 — Step 1
   npz directory = D:\data\npz
   Binary output = D:\data\binary
   Convert All Cases
   → 输出：D:\data\binary\case_0001_act_av.bin ... case_0100_atn_av.bin
        ↓

④ Simulation 页面 — Step 2+3
   simind.exe（自动） / ge870_czt.smc（自动）
   Output dir = D:\data\simind_out
   Run SIMIND Now
   → 输出：D:\data\simind_out\case_0001.a00 ... case_0100.a00
        ↓

⑤ Results → SIMIND Output 标签
   Load .a00 → 验证投影图和 sinogram
        ↓

⑥ 训练数据配对（手动/脚本）
   输入：case_XXXX.a00  → SPECT 投影 sinogram（网络输入）
   标签：case_XXXX.npz["activity"]  → 干净活度图（ground truth）
   辅助：case_XXXX.npz["mu_map"]    → 衰减图
   分割：case_XXXX_meta.json        → 肿瘤位置/尺寸/类型
```

## 12. 常见问题排查

### 启动相关

| 问题 | 解决方法 |
|:---|:---|
| `ModuleNotFoundError: PyQt6` | `pip install -r requirements.txt` |
| 窗口一闪而过 | 用 `python main.py` 查看终端错误 |
| 3D Surface 显示 "Install scikit-image" | `pip install scikit-image` |

### Phantom 生成相关

| 问题 | 可能原因 | 解决方法 |
|:---|:---|:---|
| Left ratio 显示 0% | 旧版缓存，重启软件 | 重启后应正常（二分法已修复方向） |
| 肿瘤在切片中不明显 | Overlay 遮盖 / 切片不在肿瘤位置 | 关闭 Liver Overlay，用滑块找肿瘤所在切片 |
| 路径不生效，输出到 syn3d | 旧版 bug，已修复 | 更新代码后重启 |

### 格式转换相关

| 问题 | 可能原因 | 解决方法 |
|:---|:---|:---|
| 目录选后无文件数显示 | 目录中无 case_*.npz | 确认选的是含 .npz 文件的目录 |
| .bin 大小不是 8,388,608 字节 | 矩阵不是 128 | 检查 Phantom 页 Matrix 设置 |

### SIMIND 相关

| 问题 | 原因 | 解决方法 |
|:---|:---|:---|
| 路径解析错误（`WARNING: Converting switch /U\...`）| 绝对路径含反斜杠被 SIMIND 误解 | 新版 bat 已用 `pushd` + 相对路径解决，重新生成 bat |
| `czt.cr4: file not found` | SIMIND 安装不完整 | 重新下载完整安装包 |
| SIMIND 立即退出 | .smc 格式错误 | 确认 `ge870_czt.smc` 以 `SMCV2` 开头 |
| `.a00` 文件全为零 / SIMIND Output 中 Total≈0 | 活度图值太小 | 将 Total counts 从 8.0×10⁴ 增大到 **1000.0×10⁴**（=10M） |
| `.a00` 大小不是 3,932,160 字节 | 投影参数不匹配 | 检查 .smc 中 Index-29（投影数=60）和 Index-76/77（矩阵=128）|

| 投影图全黑 | Total counts = 0 | 参见上方"活度图值太小"条目 |
