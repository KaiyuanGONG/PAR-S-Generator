## 1. 启动软件

```bash
# 激活虚拟环境
.venv\Scripts\activate

# 启动
python main.py
```

首次启动约需 10–30 秒（PyQt6 + NumPy 加载）。

软件启动时自动：
- 按上次保存的**主题**（深色/浅色）加载样式
- 按上次保存的**语言**（英/中）显示界面
- 检测 `simind/simind.exe` → 自动填入 Simulation 页面
- 检测 `simind/ge870_czt.smc` → 自动填入 .smc 路径

## 2. 快速验证流程（首次使用）

### 步骤 1：验证体模生成（约 3–5 秒）

1. 启动后默认停在 **Phantom** 页面
2. 保持默认参数，点击 **"⬡ Preview Single Case"**
3. 右侧出现三平面切片图，底部信息栏应显示：
   - Liver Vol.：900–1900 mL
   - Left Ratio：22%–48%
   - Perfusion mode：非空字符串

> **提示**：将鼠标悬停在任意参数控件上，会弹出说明 ToolTip（包含参数含义和参考数值范围）。

### 步骤 2：快速开始批量生成

1. 设置 **Output directory** 为目标路径（如 `D:\data\syn3d`）
2. 设置 **Number of cases = 1**（先测试单个）
3. 点击 Phantom 页面的 **"▶ Start Batch"** 按钮

   > 软件**自动导航到 Results 页面并立即开始生成**，无需手动切换。

4. 完成后在输出目录看到 `case_0001.npz` + `case_0001_meta.json`

> **编号从 1 开始**（`case_0001`），这是设计行为。

### 步骤 3：格式转换（约 1 秒/case）

1. 切换到 **Simulation 页面**
2. Step 1 — npz directory：选择含 `.npz` 文件的目录（选后立即显示文件数）
3. Binary output：指定输出目录（如 `D:\data\binary`）
4. 点击 **"Convert All Cases"**
5. 验证：`case_0001_act_av.bin` 和 `case_0001_atn_av.bin`，各 **8,388,608 字节**（8 MB）

### 步骤 4：SIMIND 仿真（约 4 分钟/case）

1. Step 2：确认 simind.exe 和 ge870_czt.smc 已自动填入
2. 设置 Output dir（如 `D:\data\simind_out`）
3. 点击 **"▶ Run SIMIND Now"**
4. 右侧 **Console** 标签页实时显示 SIMIND 输出
5. 仿真完成后，右侧自动切换到 **SIMIND Preview** 标签页并加载第一个 `.a00` 文件

### 步骤 5：验证仿真结果

仿真完成后，Simulation 页面右侧 **SIMIND Preview** 标签页会自动显示第一个 case 的投影和 sinogram。

- 拖动滑块浏览 60 个投影角度
- Sinogram 应显示类似弦状弧带（均匀肝脏活度 → 宽弧带；点源 → 正弦曲线）

也可在 Results → **SIMIND Output** 标签页手动加载任意 `.a00` 文件。

## 4. Phantom 页面 — 体模参数配置与预览

### 4.1 左侧参数面板

> 所有参数控件均有 **ToolTip 提示**，鼠标悬停即可查看含义和参考范围。

#### VOLUME

| 参数 | 默认值 | 说明 |
|:---|:---|:---|
| Matrix (NxNxN) | 128 | 体素矩阵边长，必须与 .smc 中 Index-76/78/81 一致 |
| Voxel size (mm) | 4.42 | GE NM/CT 870 标准值，与 .smc Index-28/31 一致 |

#### LIVER GEOMETRY

| 参数 | 默认值 | 说明 |
|:---|:---|:---|
| Scale jitter | 0.10 | 肝脏大小随机变化幅度（±10%） |
| Rotation jitter (°) | 5.0 | 旋转随机变化 |
| Global shift range | 0.05 | 整体位移随机范围（~1.4 cm） |
| Target left ratio | 0.35 | 目标左叶体积比例（二分法精确控制） |
| Smoothing σ (px) | 1.2 | 肝脏轮廓高斯平滑（0=锐边，1.0=真实） |

#### TUMORS

| 参数 | 默认值 | 说明 |
|:---|:---|:---|
| Min tumors | **1** | 每个 case 至少包含 1 个肿瘤（保证有监督学习标签） |
| Max tumors | 5 | 每个 case 最多肿瘤数 |
| Contrast min (T/L) | **2.0** | 最低肿瘤/正常肝脏活度比（Ho et al. 1997 HCC 数据） |
| Contrast max (T/L) | 8.0 | 最高肿瘤/正常肝脏活度比，每个肿瘤独立采样 |

肿瘤形态随机选择（权重 7:3）：
- **ellipsoid**（椭球体，70%）— 规则圆润
- **spiculated**（毛刺体，30%）— 表面不规则，更接近真实

肿瘤大小分布（参考 AAPM-RT 数据集）：
- 10–20 mm：45%
- 20–40 mm：40%
- 40–60 mm：15%

#### ACTIVITY

| 参数 | 默认值 | 说明 |
|:---|:---|:---|
| Total counts (×10⁴) | 8.0 | 活度图总计数（=80,000）|
| PSF σ (px) | 2.5 | UI 保留但不生效，PSF 由 SIMIND 内部处理 |
| Residual BG | 0.05 | 非特异性本底比例（0.05–0.15 为典型范围） |

> **关于 Total counts**：若 SIMIND 仿真后 .a00 中计数为 0，需将此值提高至 **1000.0×10⁴**（=10M），使活度图有足够强度。

灌注模式（每 case 随机采样）：

| 模式 | 概率 | 说明 |
|:---|:---|:---|
| Whole Liver | 5% | 全肝均匀灌注 |
| Tumor Only | 25% | 仅肿瘤区域有活度 |
| Left Only | 35% | 仅左叶灌注（肿瘤必须在左叶） |
| Right Only | 35% | 仅右叶灌注（肿瘤必须在右叶） |

#### BATCH GENERATION

| 参数 | 默认值 | 说明 |
|:---|:---|:---|
| Number of cases | 10 | 批量 case 数 |
| Global seed | 42 | 随机种子（勾选 Use fixed seed 时生效） |
| Output directory | output/syn3d | .npz 文件保存目录 |

### 4.2 按钮

| 按钮 | 功能 |
|:---|:---|
| **⬡ Preview Single Case** | 生成单个预览（不保存到磁盘），用于快速验证参数 |
| **▶ Start Batch** | **自动跳转到 Results 页面并立即开始批量生成**（推荐工作流） |
| **Save Config** | 保存当前参数为 JSON |
| **Load Config** | 从 JSON 恢复参数 |

### 4.3 右侧预览区

**Multi-Plane 标签**：Axial (Z) / Coronal (Y) / Sagittal (X) 三平面切图 + 右下信息面板

| 信息面板字段 | 含义 |
|:---|:---|
| Volume shape | 矩阵尺寸 |
| Voxel size | 固定 4.42 mm |
| Liver volume | 肝脏体积 (mL)，正常范围 900–1900 mL |
| Left lobe ratio | 二分法分割结果（应在 22%–48% 之间） |
| No. of tumors | 实际植入肿瘤数（≥1） |
| Perfusion mode | 本次随机选择的灌注模式 |
| Total counts | 活度图总计数 |

**3D Surface 标签**：蓝色透明体 = 肝脏，红色 = 肿瘤（需安装 scikit-image）

**Channel 切换**：Activity Map / μ-map (Attenuation)

**Liver Overlay**：开启时绿色高亮肝脏范围。查看肿瘤时建议**关闭 Overlay**，肿瘤对比度更清晰。

## 5. Results 页面 — 批量生成与统计

### 5.1 操作

**推荐方式**：在 Phantom 页面设置好参数后，点击 **"▶ Start Batch"** 直接跳转并启动，无需手动切换页面。

**手动方式**：切换到 Results 页面 → 点击 **"▶ Start Batch Generation"**（参数从 Phantom 页面实时读取）。

### 5.2 进度区

- 进度条：已完成/总数，百分比
- ETA 和 Elapsed 实时更新
- Case Table 实时追加行

### 5.3 统计卡片

| 卡片 | 含义 |
|:---|:---|
| Completed | 已完成/总数 |
| Liver Vol. Mean | 平均肝脏体积 (mL)，期望值 1100–1500 mL |
| Left Ratio Mean | 平均左叶比例（期望值接近 Target left ratio 设置值） |
| Avg Tumors/Case | 平均每 case 肿瘤数 |
| Total Tumors | 所有 case 总肿瘤数 |
| Avg Gen. Time | 平均每 case 耗时 (s) |

### 5.4 标签页

| 标签 | 内容 |
|:---|:---|
| **Statistics Charts** | 6 张分布图（肝体积、左叶比例、肿瘤数、肿瘤直径、灌注模式、生成时间） |
| **Case Table** | 每 case 详细数据（Case ID 从 0001 开始、种子、肿瘤直径等） |
| **Log** | 生成日志 |
| **SIMIND Output** | .a00 投影可视化，可手动加载任意文件（见第 7 节）|

### 5.5 输出文件

```
output/syn3d/
├── case_0001.npz          ← 体模数据
├── case_0001_meta.json    ← 元数据（肿瘤参数）
├── case_0002.npz
├── ...
└── batch_summary.json     ← 统计汇总
```

## 6. Simulation 页面 — 格式转换与 SIMIND 仿真

### 6.1 STEP 1 — FORMAT CONVERSION (npz → .bin)

| 字段 | 说明 |
|:---|:---|
| npz directory | 含 `case_XXXX.npz` 的目录 |
| Interfile output | 输出 `.bin` 文件目录（选目录后立即显示找到的文件数） |
| Voxel size | 应与 Phantom 页面一致（默认 4.42 mm） |

点击 **"Convert All Cases"** 后输出：

```
output/binary/
├── case_0001_act_av.bin   ← 活度图（SIMIND 通过 /FS:case_0001 读取）
├── case_0001_atn_av.bin   ← 衰减图（SIMIND 通过 /FD:case_0001 读取）
├── case_0002_act_av.bin
└── ...
```

文件格式：float32，C-order (Z,Y,X)，无头文件，大小恒为 **8,388,608 字节**（128³×4）。

> **命名约定**：后缀 `_av` 是 SIMIND XcatBinMap（Index=-7）的强制要求，不可更改。

### 6.2 STEP 2 — SIMIND CONFIGURATION

| 字段 | 说明 |
|:---|:---|
| simind.exe | 启动时自动检测 `simind/simind.exe` |
| .smc config | 自动检测 `simind/ge870_czt.smc` |
| Output dir | SIMIND 输出目录（`.a00`、`.mhd`、`.res`、`.spe` 等） |
| Photon histories | 仅显示用，实际光子数由 .smc 中 Index-26 控制 |

### 6.3 STEP 3 — GENERATE & RUN

| 按钮 | 功能 |
|:---|:---|
| **Generate .bat Script** | 生成脚本后先检查内容再运行 |
| **▶ Run SIMIND Now** | 自动生成脚本后立即运行 |
| **■ Stop** | 终止当前进程 |

**右侧标签页：**

| 标签 | 内容 |
|:---|:---|
| **Console** | 实时输出：蓝=INFO，绿=OK，红=ERROR，橙=WARN |
| **SIMIND Preview** | 仿真完成后**自动切换到此标签**并加载第一个 `.a00` 文件的投影/sinogram |

**生成的 .bat 脚本逻辑：**
```batch
# 将 .smc 复制到 binary 目录（用相对路径 /FS:case_0001 避免 SIMIND 路径解析问题）
copy "simind\ge870_czt.smc" "output\binary\" /Y > nul

pushd "output\binary"
simind.exe ge870_czt "D:\output\simind\case_0001" /FS:case_0001 /FD:case_0001
popd
```

**关键约束**：
- `/FS:` 和 `/FD:` 只传 stem（`case_0001`），**不加路径、不加 `_act_av`**
- SIMIND 自动在当前目录找 `case_0001_act_av.bin` 和 `case_0001_atn_av.bin`

**SIMIND 正常输出文件（每个 case）：**

| 文件 | 说明 |
|:---|:---|
| `.a00` | 主窗投影（float32，60×128×128 = 3,932,160 字节） |
| `.mhd` | MetaImage 格式头文件 |
| `.res` | 仿真摘要统计 |
| `.spe` | 能谱数据 |

## 7. SIMIND Output 可视化

**两个入口**（内容完全一致）：
- **Simulation 页面** → 右侧 **SIMIND Preview** 标签：仿真后自动加载第一个 `.a00`
- **Results 页面** → **SIMIND Output** 标签：手动加载任意 `.a00` 文件

### 7.1 加载文件

点击 **"Load .a00 File"** → 选择任意 `.a00` 文件。

自动检测维度（支持 128×128、64×64、256×256 矩阵）：

```
60 proj  |  128×128  |  Max: 1234.5  |  Total: 5.231e+06
```

如果 **Total 接近 0**，说明仿真没有产生计数——参见第 12 节排查。

### 7.2 投影查看器（左侧）

- 显示单个投影（一个角度的探测器图像）
- 拖动 **Projection 滑块** 在 1–N 个投影间切换
- 显示当前投影编号和对应角度（从 180° 开始，每步 6°）
- 支持鼠标滚轮缩放和拖拽查看细节

### 7.3 Sinogram 查看器（右侧）

- X 轴 = 探测器列位置
- Y 轴 = 投影角度
- 拖动 **Row 滑块** 选择探测器行（通常选中间行最有信息量）

| 外观 | 解释 |
|:---|:---|
| 中央呈现弧形亮带 | 正常——肝脏在旋转过程中形成的正弦形迹 |
| 完全空白/全黑 | 仿真无计数，检查活度图值和 SIMIND 配置 |
| 单条亮线 | 探测器几何错误 |
| 随机噪声点 | 统计噪声，增大 Index-26 光子数可改善 |

```

   完成后：Statistics Charts 显示分布图
   → 输出：D:\data\npz\case_0001.npz ... case_0100.npz
           D:\data\npz\batch_summary.json
        ↓

③ Simulation 页面 — Step 1
   npz directory = D:\data\npz
   Binary output = D:\data\binary
   Convert All Cases
   → 输出：D:\data\binary\case_0001_act_av.bin / case_0001_atn_av.bin ...
        ↓

④ Simulation 页面 — Step 2+3
   simind.exe（自动） / ge870_czt.smc（自动）
   Output dir = D:\data\simind_out
   Run SIMIND Now
   → 仿真完成，自动切换到 SIMIND Preview 标签
   → 自动加载 case_0001.a00 显示投影和 sinogram
        ↓

⑤ 训练数据配对（手动/脚本）
   输入：case_XXXX.a00   → SPECT 投影（网络输入）
   标签：case_XXXX.npz["activity"]  → 干净活度图（ground truth）
   辅助：case_XXXX.npz["mu_map"]    → 衰减图
   分割：case_XXXX_meta.json        → 肿瘤位置/尺寸/形态
```

## 12. 常见问题排查

### 启动相关

| 问题 | 解决方法 |
|:---|:---|
| `ModuleNotFoundError: PyQt6` | `pip install -r requirements.txt` |
| 窗口一闪而过 | 用 `python main.py` 查看终端错误 |
| 3D Surface 显示 "Install scikit-image" | `pip install scikit-image` |

### Phantom 生成相关

| 问题 | 可能原因 | 解决方法 |
|:---|:---|:---|
| Left ratio 显示 0% | 旧版缓存 | 重启软件后应正常（二分法已修复） |
| 肿瘤在切片中不明显 | Overlay 遮盖 / 切片不在肿瘤位置 | 关闭 Liver Overlay，用滑块找肿瘤所在切片 |
| 所有 case 肿瘤数=0 | Min tumors 设为 0 | 将 Min tumors 设为 1（默认值） |

### 格式转换相关

| 问题 | 可能原因 | 解决方法 |
|:---|:---|:---|
| 目录选后无文件数显示 | 目录中无 case_*.npz | 确认选的是含 .npz 文件的目录 |
| .bin 大小不是 8,388,608 字节 | 矩阵不是 128 | 检查 Phantom 页 Matrix 设置 |

### SIMIND 相关

| 问题 | 原因 | 解决方法 |
|:---|:---|:---|
| 路径解析错误 | 绝对路径含反斜杠被 SIMIND 误解 | 新版 bat 已用 `pushd` + 相对路径解决，重新生成 bat |
| `czt.cr4: file not found` | SIMIND 安装不完整 | 重新下载完整安装包 |
| SIMIND 立即退出 | .smc 格式错误 | 确认 `ge870_czt.smc` 以 `SMCV2` 开头 |
| `.a00` 全为零 / Total≈0 | 活度图值太小 | 将 Total counts 从 8.0×10⁴ 增大到 **1000.0×10⁴**（=10M） |
| `.a00` 大小不是 3,932,160 字节 | 投影参数不匹配 | 检查 .smc 中 Index-29（投影数=60）和 Index-76/77（矩阵=128）|

### SIMIND Preview / SIMIND Output 可视化相关

| 问题 | 原因 | 解决方法 |
|:---|:---|:---|
| 仿真后未自动切换到 Preview | 仿真失败（exit code≠0） | 检查 Console 标签中的红色 ERROR 信息 |
| 无法检测维度 | 文件大小不是 N×M×M×4 | 手动确认 .a00 文件大小是否为标准格式 |
| 投影图全黑 | Total counts = 0 | 参见上方"活度图值太小"条目 |
| Sinogram 呈水平线 | 旋转中心偏移 | 检查 .smc 中 RotationCentre 和 StartingAngle |
