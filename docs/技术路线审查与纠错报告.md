## 已修复问题详情

### Bug-1 ✅：SIMIND 命令行参数（`interfile_writer.py`）

**修复前（错误）：**
```batch
simind.exe czt_ge.smc /FI:"case_0001_act_1.h33" /FA:"case_0001_atn_1.h33" /FO:"output\case_0001" /NN:5000000
```

**修复后（正确）：**
```batch
cd /d "D:\path\to\simind\"
"simind.exe" ge870_czt "D:\output\case_0001" /FS:"D:\binary\case_0001_act" /FD:"D:\binary\case_0001_atn"
```

**关键变化：**
- 第1参数 = `.smc` 基名（不含扩展名），从 smc_file 路径自动提取
- 第2参数 = 输出文件基名（绝对路径）
- `/FS:` = 活度图二进制基名（无扩展名）
- `/FD:` = 密度/衰减图二进制基名（无扩展名）
- 移除了不存在的 `/FI:`, `/FA:`, `/FO:`, `/NN:` 参数
- SIMIND 必须从 `.smc` 文件所在目录运行（已添加 `cd /d` 跳转）

### Bug-2 ✅：输出格式改为 `.bin`（原：Interfile `.h33 + .i33`）

**问题根源：** `.smc` 中 Index-14=-7（密度图类型）和 Index-15=-7（源图类型）要求 SIMIND 以原始 float32 二进制格式读取输入，不需要 Interfile 头文件。

**修复：** `write_interfile()` 函数改写为 `write_bin()`，直接写 `.bin` 文件：
```python
# 修复后
volume.astype(np.float32).tofile(str(bin_path))
# NumPy C-order (Z, Y, X) 即为 SIMIND 期望的切片优先顺序
# Index=-7 模式下无需 Interfile 头文件，也无需轴转置
```

**文件命名变化：**
- 之前：`case_0001_act_1.h33` + `case_0001_act_1.i33`
- 现在：`case_0001_act.bin`（去掉了 `_1` 后缀，与 Issue-5 一并修复）

### Issue-7 ✅：活度图 PSF 模糊（`phantom_generator.py`）

**修复：** 移除了步骤5中的 `gaussian_filter` 调用。

**原因：**
- SIMIND 通过准直器/探测器模型自动施加 PSF（`cfg.psf_sigma_px` 在物理仿真中等效于探测器参数）
- 向 SIMIND 输入已模糊的活度图 = 双重模糊
- Ground truth 应为干净的活度分布（训练目标 = 从 SPECT 恢复此分布）

**修复后代码（步骤5）：**
```python
# PSF 由 SIMIND 内部处理，Python 端只归一化 + 泊松采样
if activity.sum() > 0:
    activity = activity / activity.sum() * cfg.total_counts
    activity = rng.poisson(np.maximum(activity, 0)).astype(np.float32)
```

### Issue-10 ✅：Cantlie 分割改用二分法（`phantom_generator.py`）

**修复前：** 固定步长 ±0.01 梯度下降，最大误差 ±5%

**修复后：** 二分法，`cantlie_iter_max=12` 次迭代后精度 < 0.012%

```python
lo, hi = cfg.cantlie_offset_range[0], cfg.cantlie_offset_range[1]
for _ in range(cfg.cantlie_iter_max):
    mid = (lo + hi) / 2.0
    left_lobe, _ = Geometry3D.split_liver_lobes(liver, shape, tilt_deg=tilt, offset=mid)
    ratio = left_lobe.sum() / liver_vol
    if ratio < cfg.target_left_ratio:
        hi = mid
    else:
        lo = mid
best_offset = (lo + hi) / 2.0
```

## 当前技术路线（纠正版）

### 阶段 1：体模生成

```
PhantomConfig（参数）
    ↓
PhantomGenerator.generate_one()
    ├── 步骤1：肝脏掩码（右叶 ∪ 左叶 ∩ 穹隆 − 胆囊窝 → 高斯平滑）   ✅
    ├── 步骤2：Cantlie 二分法分割左右叶（精度 <0.012%）               ✅
    ├── 步骤3：μ-map（5层：water/lung/spine/liver/fat）               ⚠️ 横膈膜待补
    ├── 步骤4：肿瘤植入（4种形态）                                    ✅
    └── 步骤5：活度图（灌注 + 梯度 + 对比度 + 泊松，无 PSF）          ✅
        ↓
输出：case_{XXXX}.npz（activity/mu_map/masks）+ case_{XXXX}_meta.json
```

### 阶段 2：格式转换（npz → .bin）

```
case_{XXXX}.npz
    ↓
write_bin()
    ├── 读取 activity / mu_map（float32）
    ├── 直接以 C-order (Z,Y,X) 写入（无需转置，与 SIMIND Index=-7 对应）
    └── 输出 .bin 文件
        ↓
输出：case_{XXXX}_act.bin  +  case_{XXXX}_atn.bin
```

### 阶段 3：SIMIND 仿真

```
运行前确认：
    simind/
    ├── simind.exe           ← 已安装
    └── ge870_czt.smc        ← 用 change.exe 配置（见下方指南）

正确命令（.bat 自动生成）：
    cd /d "simind/"
    simind.exe ge870_czt "output\case_0001" /FS:"binary\case_0001_act" /FD:"binary\case_0001_atn"

SIMIND 输出：
    case_0001.a00   ← 主窗投影（网络输入）
    case_0001.b00   ← 低散射窗
    case_0001.h00   ← 头文件（Interfile）
```

### 阶段 4：训练数据配对

```
输入：case_{XXXX}.a00      ← SPECT 投影（含 Poisson 噪声、衰减、散射）
标签：case_{XXXX}.npz      ← Ground truth 活度图（干净，无噪声，无 PSF）
辅助：case_{XXXX}.npz      ← μ-map（衰减校正参考）
元数据：case_{XXXX}_meta.json ← 肿瘤位置/尺寸/类型
```

## ge870_czt.smc 配置指南

### 文件应放在哪里？

```
PAR-S-Generator/
└── simind/
    ├── simind.exe           ← SIMIND 可执行文件
    ├── ge870_czt.smc        ← ✅ 你配置后保存到这里
    └── README.txt
```

软件会自动在 `simind/ge870_czt.smc` 查找此文件（`simulation_page.py` 的 `_auto_detect_simind()` 逻辑）。

### 配置时什么文件作为输入输出？

`change.exe` 是 SIMIND 自带的交互式参数编辑器，**不需要输入外部文件**，直接从零创建配置：

```
1. 打开终端，进入 SIMIND 安装目录（含 change.exe 的目录）
2. 运行：change
3. 按下方参数表逐项输入
4. 主菜单选 4（Save），保存为：PAR-S-Generator/simind/ge870_czt.smc
```

**注意：** `change.exe` 会在当前目录或你指定的路径写出 SMCV2 格式的 `.smc` 文件。写完后就是 SIMIND 可直接读取的配置文件，**不需要再做任何转换**。

### Index-26（光子历史数）

Index-26 单位为**百万（million）次/投影**（v4.5+）：
- 测试用：`1.0` = 1M 次/投影，速度快
- 生产用：`10.0` = 10M 次/投影，统计噪声低
